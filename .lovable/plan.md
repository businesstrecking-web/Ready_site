# Платформа готовых решений для бизнеса

Премиальный B2B-сайт на русском языке: тёмный hero, светлые секции, sticky header с blur, карточная подача, мягкие hover-анимации. Многостраничная архитектура с заделом под CMS-сущности.

## Визуальная система

- Фон: почти белый `#FAFAFA` / `#FFFFFF`
- Тёмные секции (hero, финальный CTA): глубокий графитово-синий `#0E1424` с мягкими blue/green glow-пятнами
- Акцент основной: насыщенный синий `#2256E6`
- Акцент вторичный (CTA, результаты): зелёный `#16A34A`
- Карточки: белые, тонкая граница `#E6E8EE`, скругление 16px, тень мягкая, hover — лёгкий подъём + усиление тени
- Типографика: Inter / системный sans, плотные крупные заголовки (h1 56–72px desktop), короткие абзацы
- Много воздуха, без градиентного шума, mobile-first

## Хедер и футер

**Header (sticky, semi-transparent + backdrop-blur, тонкая нижняя граница):**

- Лого «Сборка» (рабочее название платформы)
- Навигация: Главная · Каталог решений · Кейсы · О платформе · Контакты
- CTA справа: «Подобрать решение» (синяя кнопка)
- Mobile: гамбургер → выезжающее меню

**Footer (светлый, разделён на 4 колонки):**

- Лого + краткое описание платформы
- Навигация (повтор основной)
- Контакты: Telegram, email, график
- Низ: © 2026 Сборка · Политика конфиденциальности

## Главная — UI-структура по блокам

1. **Hero (тёмный)** — заголовок «Готовые решения для бизнеса — в одном месте», подзаголовок «Соберите систему компании из отдельных инструментов и комплексных наборов под ваши задачи», CTA «Подобрать решение» + «Смотреть каталог», мягкие синие/зелёные glow-blur-пятна

## 8 примеров карточек отдельных решений

1. Должностные инструкции — описывают зоны ответственности и убирают «серые зоны»
2. База знаний — единое хранилище процессов, регламентов и материалов
3. Трекинг — недельный ритм управления приоритетами и результатом
4. Упаковка продукта — превращает продукт в понятное предложение для клиента
5. База инвесторов — структурированная воронка для привлечения капитала
6. CustDev — система проблемных интервью для проверки гипотез
7. Опросы клиентов — количественная валидация сегментов и офферов
8. KPI-система — измеримые показатели для команд и руководителей

## 4 примера наборов

1. **Порядок в бизнесе** — должностные, регламенты, база знаний, трекинг, отчётность. Результат: прозрачная управляемая компания
2. **Подготовка к инвестициям** — упаковка компании, инвестматериалы, база инвесторов, воронка раундов. Результат: готовность к питчам и сделкам
3. **Система управления командой** — роли, KPI, планёрки, мотивация, адаптация. Результат: предсказуемая работа команды
4. **Исследование клиента и усиление продукта** — CustDev, опросы, сегментация, офферы. Результат: продукт, который покупают

## 4 примера кейсов

1. Производственная компания, 120 чел. — внедрили оргструктуру, регламенты, трекинг → сокращение операционной нагрузки собственника на 60%
2. IT-сервис, B2B SaaS — упаковка для инвесторов + база инвесторов → закрыли pre-seed раунд за 4 месяца
3. Сеть розницы, 18 точек — KPI-система и недельные планёрки → +22% к среднему чеку за квартал
4. Онлайн-школа — CustDev и обновление офферов → конверсия лендинга x2.3

## Каталог решений (UX)

Hero страницы → краткая инструкция «3 сценария входа: по направлению, по задаче, по готовому набору» → панель фильтров (направление, задача, тип: решение/набор/блок, стадия бизнеса, поиск) → сетка карточек. Tabs переключают вид: «Все · Решения · Наборы».

## Шаблоны внутренних страниц

- **Направление** — заголовок, кому подходит, задачи, список решений, готовые наборы, кейсы, FAQ, CTA
- **Отдельное решение** — название, проблема, кому, что входит, результат, связанные решения, CTA
- **Набор** — название, для кого, задачи, состав решений, что получает бизнес, связанные наборы, CTA

## CMS-модели (заложены как типизированные коллекции)

```
categories  { slug, title, description, icon, order }
solutions   { slug, title, summary, problem, audience, contents[], result, categoryId, tags[], stage }
bundles     { slug, title, audience, tasks[], solutionIds[], outcome, categoryId }
cases       { slug, company, categoryId, task, implemented[], result, tags[] }
articles    { slug, title, excerpt, body, cover, tags[], publishedAt }
faq         { question, answer, scope }
contacts    { name, contact, businessType, task, comment, createdAt }
```

На первом этапе — TypeScript-сидинг в `src/data/*.ts` с теми же интерфейсами; миграция в Lovable Cloud (Postgres + RLS) — следующим шагом без перестройки UI.

## SEO-рекомендации

- Отдельные роуты для каждого направления / решения / набора → отдельные `head()` с `title`, `description`, `og:title`, `og:description`
- Семантические H1/H2, ЧПУ-слаги на кириллической транслитерации (`/catalog/poryadok-v-biznese`)
- Schema.org: `Organization` на /, `Service` для решений, `FAQPage` на главной и направлениях, `Article` для блога, `BreadcrumbList` на внутренних
- Sitemap.xml + robots.txt через server route
- Внутренняя перелинковка: решение ↔ направление ↔ набор ↔ кейс

## Что не делаем

Никакого слова «супермаркет», корзины, цен, e-commerce-карточек, кричащих градиентов и маркетингового шума.
