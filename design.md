# readyHub Design Standards

Этот документ фиксирует UI-стандарты readyHub по текущему фронтенд-коду. Его задача — сохранить единую визуальную философию при создании новых страниц, блоков и компонентов.

## 1. Дизайн-философия

readyHub — сдержанный B2B-интерфейс для собственников и руководителей SMB. Визуальная система должна помогать быстро понять задачу, выбрать решение и оставить заявку.

Ключевые принципы:

- Интерфейс рабочий, спокойный и доверительный, без декоративной перегрузки.
- Первый слой коммуникации — бизнес-задача, результат и формат внедрения.
- Hero и финальные CTA используют тёмную поверхность, чтобы создавать ощущение веса и фокуса.
- Основное содержимое строится через светлые карточки, сетки и короткие смысловые блоки.
- Карточка — главная единица интерфейса для категорий, решений, наборов, кейсов и статей.
- Навигация должна быть предсказуемой: каталог, кейсы, база знаний, контакты.
- Новые страницы не должны выглядеть как лендинг с рекламной графикой; они должны выглядеть как часть платформы решений.

## 2. Цветовая система

Цвета задаются через CSS variables в `src/styles.css` и используются Tailwind-токенами. Новые элементы должны брать цвета из токенов, а не из произвольных hex.

Основные токены:

- `--background: oklch(0.99 0.002 260)` — почти белая база страницы.
- `--foreground: oklch(0.18 0.02 260)` — основной тёмный текст.
- `--card: oklch(1 0 0)` — поверхность карточек.
- `--primary: oklch(0.52 0.21 264)` — главный индиго-акцент для CTA, ссылок, активных состояний.
- `--primary-foreground: oklch(0.99 0.002 260)` — текст на primary.
- `--success: oklch(0.62 0.16 150)` — зелёный акцент для результата, статуса и позитивных блоков.
- `--muted: oklch(0.97 0.004 260)` — мягкие вторичные поверхности.
- `--muted-foreground: oklch(0.5 0.02 260)` — вторичный текст.
- `--hairline: oklch(0.9 0.005 260)` — тонкие разделители и границы карточек.
- `--surface-dark: oklch(0.18 0.03 260)` — тёмные hero/CTA-секции.
- `--surface-dark-foreground: oklch(0.97 0.003 260)` — текст на тёмной поверхности.

Правила применения:

- Primary использовать для главных действий, ссылок, активных фильтров и ключевых иконок.
- Success использовать только для результата, подтверждения, выгоды или успешного статуса.
- Hairline использовать для тонких границ, чтобы интерфейс оставался лёгким.
- Тёмные поверхности использовать для PageHero, главного hero и FinalCTA.
- Не вводить новые доминирующие палитры без обновления этого документа и `src/styles.css`.

## 3. Типографика

Шрифты подключаются через Google Fonts:

- Основной текст: `Inter`.
- Заголовки и брендовые элементы: `Manrope`.
- Fallback: системные sans-serif шрифты.

Базовые правила:

- `body` использует `font-sans`.
- `h1`, `h2`, `h3`, `h4` используют `font-display`.
- Заголовки имеют `font-weight: 700`.
- Letter spacing для обычных заголовков — `0`.
- Верхние метки секций используют `uppercase`, `text-xs`, `font-medium`, `tracking-[0.18em]` или `tracking-wider`.

Типовая шкала:

- Главный hero: `text-4xl md:text-7xl`, плотный line-height.
- PageHero: `text-4xl md:text-6xl`.
- Section title: `text-3xl md:text-5xl`.
- Карточка решения: `text-lg`.
- Карточка набора: `text-2xl`.
- Body text: `text-sm`, `text-base`, для hero `md:text-lg`.

## 4. Layout и сетки

Основной контейнер — `.container-page`:

- `max-width: 1200px`.
- Mobile padding: `1.25rem`.
- Desktop padding: `2rem`.
- Центрирование через `margin-inline: auto`.

Вертикальный ритм:

- Обычные секции: `py-20 md:py-28`.
- PageHero: `py-24 md:py-32`.
- Главный hero: `py-24 md:py-36`.
- Внутренние страницы деталей: чаще `py-16`, вспомогательные секции `py-8`.
- Финальный CTA: `py-20 md:py-28`.

Сетки:

- Базовый gap карточек: `gap-5`.
- Компактные интерактивные сценарии: `gap-3`.
- Категории на главной: `md:grid-cols-2 lg:grid-cols-4`.
- Решения: `sm:grid-cols-2 lg:grid-cols-3/4`.
- Наборы: `md:grid-cols-2`.
- Кейсы: `md:grid-cols-3`.

Правило: новая страница должна сначала использовать `PageHero` или `Section`, затем `container-page`, затем сетку карточек или содержательный блок.

## 5. Радиусы, слои и разделители

Системный радиус:

- `--radius: 0.5rem` — базовые 8px.
- Карточки используют `border-radius: var(--radius)`.
- Иконки и небольшие панели могут использовать `rounded-lg` или `rounded-xl`.
- CTA-кнопки используют `rounded-full`.
- Большие декоративные панели не должны получать чрезмерные скругления.

Слои:

- Header: sticky, `z-50`, полупрозрачный фон и blur.
- Hero/CTA: `surface-dark`, `hero-glow`, `overflow-hidden`.
- Карточки: белая поверхность, hairline-border, мягкий hover.
- Footer: `bg-muted/40`, hairline-разделители.

Разделители:

- Использовать `.hairline-t` и `.hairline-b`.
- Не использовать тяжёлые borders, если достаточно hairline.

## 6. Карточки

Базовая карточка — `.card-soft`:

- Белая поверхность `var(--card)`.
- Граница `1px solid var(--hairline)`.
- Radius `var(--radius)`.
- Hover: `translateY(-3px)`, мягкая тень, лёгкое изменение border-color.

Статичная карточка — `.card-static`:

- Используется для панелей, форм, информационных блоков.
- Отключает hover-поднятие и тень.

Типовые карточки:

- `CategoryCard`: иконка, заголовок, описание, ссылка “Подробнее”.
- `SolutionCard`: тип “Отдельное решение”, заголовок, summary, audience, result, stages, tags.
- `BundleCard`: бейдж “Готовый набор”, аудитория, задачи, результат.
- `CaseCard`: компания, задача, внедрено, результат, tags.
- Article card: дата, заголовок, excerpt, tags, ссылка “Читать”.

Правила:

- В карточках держать padding `p-6`, `p-7` или `md:p-8`.
- Метки внутри карточек — `text-xs uppercase tracking-wider text-muted-foreground`.
- Теги — pill-элементы `rounded-full bg-muted px-2.5 py-1 text-xs`.
- В карточках результата использовать `success`, но не превращать всю карточку в зелёную.

## 7. Кнопки и интерактивные элементы

Primary CTA:

- `rounded-full bg-primary px-6/7 py-3/3.5 text-sm font-medium text-primary-foreground`.
- Hover чаще через `hover:opacity-90`.
- Для CTA с иконкой использовать `inline-flex items-center gap-2`.

Secondary CTA на тёмной поверхности:

- `rounded-full border border-white/20 bg-white/5`.
- `text-surface-dark-foreground`.
- `backdrop-blur`.
- Hover: `hover:bg-white/10`.

Фильтры:

- Pill-кнопки.
- Active: `bg-foreground text-background`.
- Inactive: `border border-[color:var(--hairline)] text-muted-foreground hover:text-foreground`.

Сценарные кнопки каталога:

- Используют `scenario-button card-soft card-static`.
- Hover и active подсвечиваются `primary/5`, border становится `primary`.
- Обязателен pointer cursor для интерактивных кнопок.

Ссылки:

- Основные текстовые ссылки: `text-primary hover:underline` или `hover:text-foreground`.
- Карточки-ссылки должны использовать `group` и лёгкую анимацию иконки `ArrowUpRight`.

## 8. Формы

Базовый input — `.form-input`:

- `width: 100%`.
- `min-height: 2.9rem`.
- Padding `0.85rem 1rem`.
- Border `1px solid var(--hairline)`.
- Radius `0.5rem`.
- Background `var(--background)`.
- Focus border `var(--primary)`.

Правила:

- Label: `text-xs font-medium text-muted-foreground`.
- Группы полей: `grid gap-5`.
- Двухколоночные группы включать с `md:grid-cols-2`.
- Submit-кнопка всегда primary CTA.
- После успешной отправки показывать отдельное состояние, а не alert.

## 9. Структурные компоненты сайта

`Header`:

- Sticky top header, высота `h-16`.
- До скролла: `bg-background/40 backdrop-blur-sm`.
- После скролла: `bg-background/75 backdrop-blur-md hairline-b`.
- Desktop nav: `gap-7`, muted text, active link foreground.
- Mobile nav: кнопка `Menu/X`, вертикальный список, отдельный CTA.

`Footer`:

- `hairline-t mt-24 bg-muted/40`.
- Сетка `md:grid-cols-4`.
- Брендовая метка `rH`.
- Колонки: описание, навигация, контакты, документы.

`PageHero`:

- Используется на внутренних страницах.
- Всегда `surface-dark relative overflow-hidden`.
- Внутри `hero-glow`.
- Содержит eyebrow, h1, description, optional children.

`Section`:

- Используется для повторяемых секций на главной и страницах.
- Поддерживает `eyebrow`, `title`, `description`, `dark`, `id`.
- Не создавать новый wrapper, если хватает `Section`.

`FinalCTA`:

- Завершает коммерческие страницы.
- Тёмная поверхность, центрированный текст, две кнопки.
- Используется после каталога, деталей, about, cases.

## 10. Правила добавления новых страниц

Делать:

- Начинать страницу с `PageHero`, если это самостоятельный раздел.
- Использовать `Section` для смысловых блоков.
- Использовать существующие карточки или их стиль.
- Держать тексты короткими: задача, для кого, результат, следующий шаг.
- Использовать lucide-react для иконок.
- Использовать токены Tailwind/CSS variables.
- Проверять мобильные сетки и переносы текста.

Не делать:

- Не добавлять одноразовые hex-цвета без обновления токенов.
- Не создавать декоративные блоки, которые не помогают выбору решения.
- Не делать вложенные карточки внутри карточек.
- Не использовать тяжёлые тени, крупные градиентные плашки и случайные иллюстрации.
- Не создавать новые стили кнопок, если подходят primary, secondary или filter.
- Не смешивать рыночные термины без привязки к задаче, аудитории и результату.

## 11. Быстрый чеклист перед сдачей UI

- Страница использует `readyHub` и текущую навигацию.
- Есть понятный h1, description и главный следующий шаг.
- Сетки не ломаются на mobile.
- Все интерактивные элементы имеют hover/focus состояние.
- Цвета взяты из токенов.
- Карточки используют `card-soft` или совместимый паттерн.
- Формы используют `.form-input`.
- Тексты не выглядят как рекламный лендинг, а помогают выбрать решение.
